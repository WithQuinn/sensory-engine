## What Changed

### 1. US-101: Start a Journey (New Feature)

**Branch:** `feature/us-101-start-journey` ‚Üí PR #7

#### Architecture
`JourneyViewModel` (`@MainActor ObservableObject`) drives two independent modes:
- **`.destination`** ‚Äî SSE stream from `/api/suggest-venues`. `streamTask` holds a cancellable `Task<Void, Never>` so navigation-back cleanly cancels in-flight requests.
- **`.itinerary`** ‚Äî POST to `/api/parse-itinerary`, returns a `ParsedJourney` struct (destination, optional start/end dates, trip context, venues).

#### New Files
| File | Role |
|------|------|
| `Features/Journey/Model/Journey.swift` | SwiftData `@Model` ‚Äî parallel `[String]` arrays for venue names, descriptions, categories, relatedTo (SwiftData limitation: no nested `@Model` arrays) |
| `Features/Journey/ViewModel/JourneyViewModel.swift` | State machine ‚Äî two modes, SSE streaming, save to SwiftData |
| `Features/Journey/View/StartJourneyView.swift` | Entry ‚Äî segmented mode picker, text field / TextEditor, spinner states |
| `Features/Journey/View/JourneyVenueView.swift` | Venue list ‚Äî staggered card animations, Save Journey CTA |
| `Features/Journey/View/JourneyCompleteView.swift` | Confirmation ‚Äî animated checkmark, Done collapses full nav stack |

#### Modified Files
- `Services/TravelAPIClient.swift` ‚Äî added `parseItinerary()` method and `ParsedJourney` struct
- `Features/Home/HomeView.swift` ‚Äî "Start a Journey" card (map icon) with `@State + navigationDestination`
- `QuinnApp.swift` ‚Äî SwiftData schema updated with `Journey.self`
- `Quinn.xcodeproj/project.pbxproj` ‚Äî 5 new file refs (`AA010101‚Ä¶`), 5 build files (`AA010102‚Ä¶`), 4 new groups (`AA010103‚Ä¶`)

#### Key Decisions
- **Parallel arrays in SwiftData model** ‚Äî SwiftData (iOS 17) does not support arrays of non-`@Model` structs. `VenueSuggestion` stays as a plain `Sendable` struct used at the view layer; `Journey` stores raw arrays and reconstructs `[VenueSuggestion]` on read.
- **`@State + navigationDestination` (not `NavigationLink`)** ‚Äî Consistent with existing pattern; avoids immediate navigation on render.
- **No itinerary text stored** ‚Äî `itineraryText` is ephemeral `@Published` in ViewModel, never written to SwiftData. Privacy-first.

### 2. Quality Spike Phase 1 (7 Fixes)

**Branch:** `feature/quality-spike-phase1`

#### Fix 1 ‚Äî Permission toggle re-sync on foreground return (HIGH)
**Files:** `PermissionOnboardingViewModel.swift`, `PermissionOnboardingView.swift`

`syncTogglesToCurrentState()` was `private`, making it unreachable from the View layer. Made it `internal`. Added `@Environment(\.scenePhase)` to `PermissionOnboardingView` with:
```swift
.onChange(of: scenePhase) { _, phase in
    if phase == .active { viewModel.syncTogglesToCurrentState() }
}
```
Photos and Notifications now re-sync when the user returns from iOS Settings. Location was already correct via `CLLocationManagerDelegate`.

#### Fix 2 ‚Äî Replace GCD asyncAfter with cancellable Task (HIGH)
**File:** `Features/FirstRun/View/VenueCardView.swift`

Replaced:
```swift
DispatchQueue.main.asyncAfter(deadline: .now() + 0.2) { onDrawnTo() }
```
With:
```swift
Task {
    try? await Task.sleep(nanoseconds: 200_000_000)
    if !Task.isCancelled { onDrawnTo() }
}
```
Eliminates the last GCD call in the FirstRun feature tree. Swift 6 strict concurrency compliant.

#### Fix 3 ‚Äî guard let self in JourneyViewModel callbacks (MEDIUM)
**File:** `Features/Journey/ViewModel/JourneyViewModel.swift`

All five `Task { @MainActor [weak self] in }` blocks now open with `guard let self else { return }`. Previously, `self?.venues.append(venue)` was a silent no-op if the ViewModel was deallocated mid-stream. Pattern now matches `FirstRunViewModel` which already did this correctly.

Blocks patched:
- `fetchVenues()` ‚Üí `onVenue`, `onComplete`, `onError`
- `parseItinerary()` ‚Üí `onSuccess`, `onError`

#### Fix 4 ‚Äî Voice clip too-short user feedback (MEDIUM)
**File:** `Features/QuickCapture/ViewModel/VoiceRecorderViewModel.swift`

The `else` branch of `stopRecording()` (clips < 0.5s) was silently discarding the clip with no feedback. Added:
```swift
errorMessage = "Clip too short ‚Äî hold to record at least half a second."
```
`errorMessage` is already `@Published` and observed by `VoiceClipStepView` ‚Äî no view changes needed.

#### Fix 5 ‚Äî Disable Save Journey when venues list is empty (MEDIUM)
**File:** `Features/Journey/View/JourneyVenueView.swift`

Added `.disabled(viewModel.venues.isEmpty)` to the Save Journey button. Prevents writing an empty `Journey` record to SwiftData if the API returns 0 venues.

#### Fix 6 ‚Äî Task.isCancelled check in SSE loop (MEDIUM)
**File:** `Services/TravelAPIClient.swift`

Added at top of `for try await line in asyncBytes.lines`:
```swift
if Task.isCancelled { break }
```
`CancellationError` is already caught at the `for try await` level, but this avoids processing one additional buffered line after the outer Task is cancelled.

#### Fix 7 ‚Äî Cancel stream on StartJourneyView disappear (MEDIUM)
**File:** `Features/Journey/View/StartJourneyView.swift`

Added:
```swift
.onDisappear { viewModel.cancelStream() }
```
Without this, navigating back while a destination SSE stream was in flight left the `Task` running until it completed or errored ‚Äî consuming network and CPU in the background.

### 4. When does Sensory Engine work land in the iOS app? (US-301)
US-301 (Review Quinn's Draft Memory) is the integration point ‚Äî **Phase 2, story 3**, gated behind US-201 (arrival data) and US-202 (voice notes).

The V4 architecture decision is **on-device first**, not SE API first:

| Path | When | What's sent |
|------|------|-------------|
| **Primary:** On-device LLM | Phase 2 | Nothing ‚Äî Llama 3.2 3B / Apple Intelligence runs ported `sensoryPrompts.ts` logic in Swift, metadata only |
| **Fallback:** SE `/api/synthesize-sense` | Only if on-device quality < 85% threshold | Metadata only (`SensoryInputSchema`) ‚Äî no photos, audio, or transcripts ever transmitted |

**What gets ported from SE to Swift for US-301:**
- `generateFallbackNarrative()` logic from `sensoryPrompts.ts` ‚Äî this becomes the primary on-device narrative engine in Swift
- `calculateTranscendenceScore()` algorithm
- `MomentSenseSchema` response shape (narratives, emotions, anchors, sensory details, atmosphere)

**Quality gate:** A persona panel evaluation of on-device output vs SE Claude output determines whether the SE API fallback path is ever used in production. That evaluation runs before US-301 ships.

**Build order dependency chain:** US-102 ‚Üí US-202 ‚Üí US-201 ‚Üí US-301
